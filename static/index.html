miniLLM Codebase Architecture Map

Here is the technical design of the model and serving runtime.

1. Input Layer

Character sequence is mapped into integers using a custom mapping schema inside `tokenizer.py`.

↓

2. Embedding Block

Transforms discrete IDs into active dense representations: `Token Embedding` + `Learned Absolute Position Embedding`.

↓

3. 4x Transformer Layers (with KV Caching)

For each layer: Pre-LN → Causal Attention → Pre-LN → MLP (GELU). If Caching is enabled: Keys and Values are fetched and concatenated from `kv_cache.py`, reducing sequence calculation from O(T²) to O(1) attention steps.

↓

4. Projector Head

Output projections tied to embedding weights map vectors back to token vocabulary logits.

↓

5. Optimization Runtime

Using `quantize_dynamic` weights are packed into INT8 coefficients, shrinking disk footprints by 75% for CPU inference.
